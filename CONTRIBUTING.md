# Contributing Guide

Thank you for your interest in contributing to the Autism Typing Tutor!

## Code of Conduct

This project follows an inclusive and respectful code of conduct. Please be kind and considerate.

## How to Contribute

### Reporting Bugs
1. Check existing issues
2. Create a detailed bug report
3. Include steps to reproduce
4. Add screenshots if applicable

### Feature Requests
1. Search existing requests
2. Describe the feature clearly
3. Explain the use case
4. Consider autism-friendly design

### Pull Requests
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Add tests
5. Update documentation
6. Submit PR with clear description

## Development Setup

```bash
git clone https://github.com/BusinessBuilders/typing-tutor.git
cd typing-tutor
npm install
npm run dev
```

## Testing

```bash
npm run test
npm run test:a11y
npm run lint
```

## Accessibility Guidelines

- WCAG 2.1 AA compliance
- Keyboard navigation
- Screen reader support
- Clear visual indicators
- Autism-friendly UI patterns
